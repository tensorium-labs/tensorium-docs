🧩 Chrome Wallet

Chrome Wallet Extension

Manage TXM directly in your browser. No local node required for normal use — the wallet connects to public Tensorium RPC endpoints and keeps keys encrypted in the extension.

Overview

The Tensorium Chrome Wallet is a Manifest V3 browser extension that lets you create and manage a TXM wallet without running a local node. Keys are generated and encrypted locally — your private key never leaves your browser.

✅ Features
  • Create or import wallet
  • View address & balance
  • Send TXM
  • Transaction history
  • Mainnet Candidate
  • Export / backup wallet file
🔒 Security
  • secp256k1 ECDSA key generation
  • Argon2id + XChaCha20-Poly1305 at rest
  • Session unlock — no repeated password
  • Compatible with txmwallet CLI
  • No server — keys stay local

Install

The extension is currently available as a developer build (unpacked). Chrome Web Store submission materials are now prepared, including listing copy, support docs, and privacy policy. Final store upload still depends on the developer dashboard step.

Step 1 — Download the extension

Clone or download the repository from GitHub:

bash
git clone https://github.com/tensorium-labs/tensorium-wallet-extension.git
cd tensorium-wallet-extension
npm install
npm run build

This produces a dist/ folder — the loadable extension package.

Step 2 — Load in Chrome

  1. Open Chrome and navigate to chrome://extensions
  2. Enable Developer mode (toggle in the top-right corner)
  3. Click Load unpacked
  4. Select the dist/ folder inside the cloned repo
  5. The Tensorium Wallet icon will appear in your Chrome toolbar
💡
Click the puzzle icon (Extensions) in Chrome to pin Tensorium Wallet to your toolbar for quick access.
🧾
For Chrome Web Store review and user support, see the hosted Privacy Policy and Wallet Support pages.

Create a Wallet

  1. Click the Tensorium Wallet icon to open the popup
  2. Click Create New Wallet
  3. Set a strong password (minimum 8 characters)
  4. Confirm your password and click Create Wallet
  5. Download your backup JSON file — this is required before you can continue
  6. Click I've Saved My Backup — Continue
⚠️
There is no recovery service. If you lose your backup file or forget your password, your funds are permanently unrecoverable. Store your backup in a safe, offline location.

Import an Existing Wallet

You can import a wallet from:

  • Private key hex — 64-character hex string from txmwallet
  • Wallet JSON file — the tensorium-wallet.json format produced by txmwallet create or the extension itself
  1. Open the popup and click Import Existing Wallet
  2. Paste your private key hex or upload a .json wallet file
  3. Set a new password (this re-encrypts the key locally)
  4. Click Import Wallet
🔄
The extension is fully compatible with the txmwallet CLI. Wallet files created by the CLI can be imported here, and vice versa.

Send TXM

  1. Unlock your wallet and open the Dashboard
  2. Click Send
  3. Enter the recipient address (starts with txm1)
  4. Enter the amount in TXM
  5. Click Review to preview the transaction
  6. Click Confirm & Send

The transaction is signed locally with your private key and broadcast to the network via the public RPC. There are no transaction fees.

To run a local node and use it with the extension:

bash
# Start local mainnet node with public RPC allowed
TENSORIUM_RPC_ALLOW_PUBLIC=1 tensorium-node mainnet-candidate rpc 0.0.0.0:33332

# In extension Settings → Custom RPC → http://localhost:33332

Security Model

Key Storage

Your private key is encrypted with Argon2id (19 456 KiB memory, 3 iterations) + XChaCha20-Poly1305 and stored in chrome.storage.local. The plaintext key is never written to disk.

Session Unlock

After entering your password, the decrypted key is held in browser memory for the duration of the session. It is cleared automatically when you lock the wallet, close the browser, or the extension suspends.

Wallet File Format

The wallet file (.json) is identical to the format produced by the txmwallet CLI:

json
{
  "version": 1,
  "address": "txm1...",
  "public_key_hex": "02...",
  "encrypted_private_key": {
    "kdf": "argon2id",
    "kdf_memory_kib": 19456,
    "kdf_iterations": 3,
    "kdf_parallelism": 1,
    "cipher": "xchacha20poly1305",
    "salt_hex": "...",
    "nonce_hex": "...",
    "ciphertext_hex": "..."
  }
}

Show Private Key

Go to Settings → Show Private Key. You will be asked to re-enter your password. The raw 64-character hex private key is then shown for one-time viewing. Store it securely — never share it.

FAQ

+ I forgot my password. Can I recover my wallet?
Only if you have your backup .json file. Import it and set a new password. If you have lost both the password and the backup, funds are unrecoverable — there is no server or recovery service.
+ Is the private key ever sent to a server?
No. Key generation, encryption, and signing all happen inside your browser. Only signed transactions are broadcast to the RPC.
+ Can I use a wallet file created by txmwallet CLI?
Yes. The extension uses the exact same wallet file format. Import it via Settings → Import Wallet → Upload .json file.
+ Why does transaction history only show received TXM?
The current history tab scans blocks for outputs matching your address. Full outgoing detection requires a dedicated indexer, which is planned for Phase 9B. Your sent transactions are still confirmed on-chain — check the explorer for full history.
+ Will this be on the Chrome Web Store?
Yes. The submission pack is ready, but final publication still depends on manual Chrome Web Store dashboard steps and review timing. Until the listing is approved, use the developer build via Load Unpacked.
