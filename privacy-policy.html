Tensorium Wallet Privacy Policy

How the Tensorium Wallet browser extension handles local wallet data, RPC traffic, and user-controlled secrets.

ℹ
Short version. Tensorium Wallet is designed as a self-custody extension. Private keys are generated, stored, and used locally inside the browser extension. The extension does not require account signup or custodial wallet registration.

What We Collect

Tensorium Wallet does not intentionally collect personal identity information such as full name, phone number, postal address, or government ID.

The extension may interact with public Tensorium RPC endpoints to read chain state and broadcast signed transactions. Those RPC interactions may necessarily expose technical request metadata such as:

  • wallet address queried for balance or UTXOs
  • signed transaction payloads that the user chooses to broadcast
  • standard web request metadata such as IP address, user agent, or timestamps as seen by the RPC host

Local Wallet Storage

The extension stores wallet data in chrome.storage.local on the user’s device. This may include:

  • encrypted wallet JSON
  • selected network (`mainnet` or custom RPC)
  • optional custom RPC URL

The private key is encrypted before storage. The extension is intended to avoid sending raw private keys to Tensorium infrastructure.

RPC Usage

By default, the extension is configured to access:

  • https://rpc.tensoriumlabs.com

These endpoints are used to:

  • check block height
  • retrieve UTXOs for the active address
  • broadcast signed transactions

If the user configures a custom RPC endpoint, then traffic is sent to that user-selected endpoint instead.

No Sale of Personal Data

Tensorium Wallet is not designed to sell user data or monetize private wallet information. The project goal is to provide a self-custody wallet for the Tensorium network, not an advertising or profile-building platform.

User Responsibility and Security

Users remain responsible for:

  • protecting their password
  • backing up wallet JSON files securely
  • verifying recipient addresses before sending TXM
  • understanding that blockchain transactions are irreversible once confirmed

If a device is compromised, malware or browser compromise may still put local wallet data at risk. No software wallet can remove all endpoint risk.
